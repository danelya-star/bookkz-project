\section*{ВВЕДЕНИЕ}
\addcontentsline{toc}{section}{ВВЕДЕНИЕ}

В современных условиях цифровизации общества онлайн-сервисы развиваются стремительными темпами. Одним из наиболее востребованных направлений стала разработка мультисервисных платформ, объединяющих бронирование различных видов услуг в едином удобном интерфейсе. Потребители все больше ценят свое время и предпочитают организовывать поездки, отдых и досуг через единые экосистемы. В связи с этим, разработка локальной платформы бронирования услуг (отели, рестораны, мероприятия, транспорт) является высокоактуальной задачей, позволяющей повысить удобство для пользователей на рынке Казахстана.

\textbf{Актуальность выбранной темы} определяется растущей потребностью в централизованных онлайн-сервисах на локальном рынке, стремлением пользователей к быстрому получению информации, а также необходимостью цифровизации малого и среднего бизнеса.

\textbf{Целью курсового проекта} является разработка масштабируемой онлайн-платформы «BookKZ» для автоматизации процессов бронирования услуг.

\textbf{Задачи исследования:}
1. Провести анализ предметной области и изучить существующие системы онлайн-бронирования.
2. Спроектировать клиент-серверную архитектуру приложения и структуру базы данных.
3. Разработать функционал клиентской и серверной частей (авторизация, каталог, бронирование, личный кабинет, панель администратора).
4. Протестировать разработанный программный продукт и оценить результаты.

\textbf{Объект исследования} — процесс онлайн-взаимодействия между поставщиками услуг и потребителями.
\textbf{Предмет исследования} — программные средства и методы разработки full-stack веб-приложений.

\section{ТЕОРЕТИЧЕСКАЯ ЧАСТЬ}

\subsection{Общие понятия Web-приложений}
В современном мире веб-приложение — это клиент-серверное программное обеспечение, в котором клиент взаимодействует с веб-сервером через браузер. В рамках данного проекта разрабатывается платформа «BookKZ», предоставляющая интерфейс для бронирования отелей, столиков в ресторанах, билетов на мероприятия и услуг трансфера. Система включает пользовательскую и административную роли, обеспечивая изолированный и безопасный доступ к данным.

\subsection{Обзор существующих программных решений}
На сегодняшний день на рынке представлено множество глобальных систем бронирования (Booking.com, Airbnb, Яндекс.Афиша, Aviata). Однако многие из них имеют узкую специализацию (только отели, или только билеты). Создание единой локальной платформы, объединяющей различные виды туристических и развлекательных услуг с поддержкой местных платежных шлюзов, предоставляет значительное конкурентное преимущество.

\subsection{Основные этапы разработки Web-приложения}
Процесс создания платформы BookKZ включает следующие этапы:
1. Сбор и анализ требований (определение функционала: поиск, фильтрация, корзина, рейтинги).
2. Проектирование (UI/UX дизайн, проработка архитектуры БД MERN стека).
3. Разработка бекенда (Node.js, Express, MongoDB).
4. Разработка фронтенда (React.js, маршрутизация, внедрение адаптивного дизайна).
5. Тестирование и деплой на хостинг.

\subsection{Анализ программных средств, используемых при разработке}
Для реализации курсового проекта был выбран стек технологий MERN:
- \textbf{MongoDB}: нереляционная СУБД, отлично подходящая для гибких структур данных (отели, рестораны).
- \textbf{Express.js и Node.js}: технологии серверной части, обеспечивающие высокую производительность за счет асинхронной обработки запросов.
- \textbf{React.js}: библиотека для создания пользовательских интерфейсов, обеспечивающая плавную навигацию за счет концепции Single Page Application (SPA).

\section{ПРОЕКТИРОВАНИЕ ПРОГРАММНОГО ПРОДУКТА}

\subsection{Разработка структуры Web-приложения}
Архитектура «BookKZ» базируется на принципах REST API. 
Клиентская часть разделена на страницы:
- Главная (информация и категории).
- Каталог услуг с продвинутой фильтрацией.
- Страница детализированной информации об услуге с отзывами.
- Личный кабинет пользователя (управление бронированиями, скачивание чеков PDF).
- Панель администратора (управление пользователями, графики доходов).

\subsection{Описание средств проектирования систем}
Для проектирования системы применялись UML-диаграммы (Use Case для описания прецедентов, ER-диаграммы для описания связей в базе данных). Основными сущностями базы данных являются: Пользователь (User), Услуга (Service), Бронирование (Booking), Отзыв (Review).

\subsection{Средства разработки и окружение программирования}
В качестве среды разработки (IDE) использовался Visual Studio Code. Для тестирования запросов к серверной части применялся Postman. Контроль версий осуществлялся с помощью системы Git. Дизайн-система построена на нативном CSS с использованием переменных (CSS Custom Properties).

\section{ПРАКТИЧЕСКАЯ ЧАСТЬ (РЕАЛИЗАЦИЯ И ТЕСТИРОВАНИЕ)}

\subsection{Разработка серверной части (Backend)}
Было создано безопасное REST API. Аутентификация реализована с помощью технологии JSON Web Tokens (JWT). Пароли шифруются при помощи библиотеки \textit{bcryptjs}. В контроллерах были описаны агрегационные функции MongoDB для получения сложной статистики (выручка по месяцам, загруженность категорий).

\subsection{Разработка клиентской части (Frontend)}
На стороне клиента реализована отзывчивая верстка. Использованы хуки React (useState, useEffect, useContext) для управления состояниями и авторизацией. Для генерации квитанций об оплате (PDF) была успешно интегрирована библиотека \textit{jspdf} и \textit{jspdf-autotable}.

\subsection{Тестирование и результаты}
Проведено модульное и интеграционное тестирование. Навигация, фильтры и система бронирования работают корректно. Ошибки обработки платежей корректно перехватываются сервером, а пользователю выводятся соответствующие уведомления.

\section*{ЗАКЛЮЧЕНИЕ}

В ходе выполнения курсового проекта была успешно достигнута поставленная цель — разработана полнофункциональная веб-платформа для бронирования услуг «BookKZ».

Задачи, сформулированные во введении, последовательно выполнены:
Во-первых, был проведен анализ рынка туристических и досуговых услуг, выявлены основные проблемы существующих сервисов.
Во-вторых, спроектирована устойчивая архитектура на базе современных веб-стандартов. Разработана база данных, способная эффективно хранить информацию об отелях, пользователях, транзакциях и отзывах.
В-третьих, реализован полноценный функционал: от регистрации и поиска до имитации оплаты и формирования отчетов для администратора. 
В-четвертых, платформа прошла тестирование, результаты которого подтвердили ее высокую отзывчивость и надежность реализованного API.

Созданная платформа имеет высокую практическую ценность. Она может быть внедрена на региональном уровне в Республике Казахстан для облегчения процесса бронирования. Дальнейшее развитие проекта предполагает внедрение реального эквайринга (подключение платежных шлюзов Halyk Bank или Kaspi), добавление мультиязычности и разработку мобильного приложения.

    \textbf{Демонстрация интерфейса системы}

Данное приложение включает фрагмент кода, отвечающего за генерацию PDF-отчета (квитанции о бронировании), который был разработан в ходе выполнения практической части курсового проекта.

\vspace{0.5cm}

\begin{verbatim}
import { jsPDF } from 'jspdf';
import autoTable from 'jspdf-autotable';

export const generateBookingReceipt = (booking) => {
    const doc = new jsPDF();
    const { _id, service, guests, totalPrice } = booking;

    doc.setFillColor(99, 102, 241);
    doc.rect(0, 0, 210, 40, 'F');
    doc.setTextColor(255, 255, 255);
    doc.setFontSize(24);
    doc.text('BookKZ - Receipt', 20, 25);
    
    autoTable(doc, {
        startY: 90,
        head: [['Service', 'Guests', 'Amount']],
        body: [[ service.title, guests, `${totalPrice} KZT` ]],
    });

    doc.save(`receipt-${_id}.pdf`);
};
\end{verbatim}
